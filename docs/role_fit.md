# Role Fit

This proof system is designed to demonstrate fit for teams that need rigorous model behavior evaluation, calibrated human judgment, high-quality human data protocols, and clinical-safety domain sensitivity.

## Role families

| Role family | Evidence in this repo |
|---|---|
| Evaluation Systems Engineer | Structured data, scoring schema, validation scripts, reproducible summaries, and evidence architecture. |
| Model Evaluation / Reliability Specialist | Pairwise ranking, failure labels, audit rationales, and dimension-level scoring. |
| Human Data Quality and Rubric Design | Rubric anchors, cap rules, reviewer-operable criteria, disagreement protocol, and score consistency metrics. |
| Clinical AI Safety Evaluation | High-risk synthetic probes, escalation handling, false reassurance detection, uncertainty and scope control. |
| Model Behavior Analyst | Failure taxonomy, reasoning-faithfulness review, context-missingness analysis, and unsupported-claim detection. |
| Applied Safety Evaluation Operations | Governance checklist, limitations, non-clinical scope, reproducible runbook, and charted summaries. |

## Differentiating signal

The repository combines healthcare-domain judgment with evaluation-system structure. It shows more than domain knowledge and more than a code sample: it shows how ambiguous model behavior can become calibrated human judgment data.

## Positioning boundary

The project does not claim clinical deployment readiness, medical authority beyond Tarek's stated background, production benchmark validity, or access to proprietary model outputs. Its value is evaluation design, reviewer calibration, and safety-relevant model behavior analysis.
